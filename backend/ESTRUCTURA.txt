✅ ENDPOINTS IMPLEMENTADOS (DÍA 1)
================================================================================

1. POST /api/v1/auth/login
   ├─ Input: { email, password }
   ├─ Output: { access_token, token_type: "bearer", expires_in: 1800 }
   ├─ Status: 200 OK o 401 Unauthorized
   └─ Hash: bcrypt (factor 12)

2. POST /api/v1/auth/register
   ├─ Input: { email, password, nombre }
   ├─ Output: { id, email, nombre, is_active, roles: [] }
   ├─ Status: 201 Created o 422 Unprocessable Entity
   └─ Rol: CLIENT asignado automáticamente

3. GET /api/v1/auth/me
   ├─ Auth: Bearer token (HTTP Authorization header)
   ├─ Output: { id, email, nombre, is_active, roles: [] }
   ├─ Status: 200 OK o 401 Unauthorized
   └─ Deps: get_current_user (inyectada)

Endpoints adicionales:
- GET / (root health check)
- GET /health (para load balancers)
- GET /docs (Swagger OpenAPI)
- GET /redoc (ReDoc)

   ├─ Algorithm: HS256
   ├─ Expiration: 30 minutos (configurable en .env)
   ├─ Payload: { sub: user_id, exp, iat }

   ├─ Cost factor: 12 (SECURE)
   └─ verify_password: timing-safe comparison

✅ Autenticación
   ├─ HTTP Bearer token en header Authorization
   ├─ Validación de token en cada request autenticado
   ├─ Decodificación segura de JWT
   └─ Manejo de expiración y errores

✅ CORS Middleware
   ├─ Origins: configurable en .env
   ├─ Credentials: true
   ├─ Methods: * (configurable)
   └─ Headers: * (configurable)

✅ Validación Pydantic
   ├─ Email: EmailStr (formato válido, único en BD)

🚀 INSTRUCCIONES DE SETUP
================================================================================

OPCIÓN 1: Setup Local (Recomendado para desarrollo)
────────────────────────────────────────────────────

1. PostgreSQL en Docker
   $ docker-compose up -d
   (espera 5-10 segundos)

2. Instalar dependencias
   $ python -m venv venv
   $ source venv/bin/activate  # Windows: venv\Scripts\activate
   $ pip install -r requirements.txt

3. Crear tablas y seed
   $ python -c "from app.db.base import create_all_tables; create_all_tables()"
   $ python -c "from app.db.seed import populate_seed_data; populate_seed_data()"
   
   O usar scripts:
   $ bash run.sh              # Linux/Mac
   $ run.bat                  # Windows

4. Iniciar servidor
   $ uvicorn app.main:app --reload

5. Acceder
   http://localhost:8000/docs         # Swagger UI
   http://localhost:8000/health       # Health check

OPCIÓN 2: Verificación Rápida (sin venv)
─────────────────────────────────────────

$ docker-compose up -d
$ pip install -r requirements.txt

       "password": "admin123"
     }'

     "access_token": "eyJhbGc...",

     "expires_in": 1800

     "id": 1,
     "email": "admin@foodstore.local",
     "nombre": "Administrador",
     "is_active": true,
     "roles": []
   }

     -H "Content-Type: application/json" \
     -d '{

       "password": "password123",
       "nombre": "New User"
     }'
   
   Response (201 Created):
   {
     "id": 2,
     "email": "user@example.com",
     "nombre": "New User",
     "is_active": true,
